Klasy

Klasy w Javascript zostały wprowadzone w ECMAScript 2015 jako lukier składniowy (ang. syntactic sugar) dla istniejącego, oparte na prototypach. 

class Circle {
	constructor(radius)	{
		this.radius = radius;
	}
	
	set radius(radius) {
		this.radius = radius;
	}
	
	get circumference() {
		return 2*Math.PI*this.radius;
	}
}

Promise

Promise jest specjalnym obiektem reprezentującym wynik asynchronicznej operacji (np. zapytania do bazy danych).

Można go rozumieć jako obietnicę otrzymania wyniku po zakończeniu asynchronicznej operacji.

fetch

Fetch API jest interfejsem pozwalającym na asynchroniczne pobieranie zasobów.

Fetch w najprostszym wywołaniu przymuje URL i wywoła na nim zapytanie typu GET.

fetch('https://example.com/posts/1');
Fetch zwraca obiekt typu Promise, czyli obietnicę która w nieokreślonej przyszłości będzie zawierała odpowiedź z serwera.
const promise = fetch('https://example.com/posts/1');
W celu uzyskania odpowiedzi z serwera możemy użyć słowa kluczowego await.

const resultOfPromise = await fetch('https://example.com/posts/1');

W miejscu użycia słowa kluczowego await kod będzie "czekał" z dalszym wykonywaniem kodu aż serwer zwróci odpowiedź.
Słowa kluczowego await możemy używać tylko w funkcjach asynchronicznych!

async function asynchronousFunction() {
  const resultOfPromise = await fetch('https://example.com/posts/1');
}
Wynikiem promisa nie jest bezpośrenio odpowiedź serwera, ponieważ jest ona zapakowana w specjalny obiekt.

Jedną z metod tego obiektu jest metoda json(), która dopiero dostarczy nam dane które zwrócił serwer.
Metoda json() jest również asynchroniczna (ponieważ może być czasochłonna).

async function asynchronousFunction() {
  const resultOfPromise = await fetch('https://example.com/posts/1');
  const resultOfRequest = await resultOfPromise.json();
  consle.log(resultOfRequest);
}
Wszystko co zwracamy w funkcjach asynchronicznych zostanie automatycznie zapakowane w Promise.
Aby obsłużyć wynik asynchronicznej funkcji musimy obłużyć Promise, a więc znowu możemy zrobić to tylko w funkcji asynchronicznej.

async function fetchData() {
  const resultOfPromise = await fetch('https://example.com/posts/1');
  const resultOfRequest = await resultOfPromise.json();
  return resultOfRequest;
}

async function anotherAsynchronousFunction() {
  const data = await fetchData();
  consle.log(data);
}
Obsługa błedów - np. Internal Server Error w przypadku buga na backendzie

async function fetchData() {
  try {
    const resultOfPromise = await fetch('https://example.com/posts/1');
    const resultOfRequest = await resultOfPromise.json();
    return resultOfRequest;
  } catch (e) {
    console.log("Error occurred: ", e);
    return {};
  }
}

async function anotherAsynchronousFunction() {
  const data = await fetchData();
  consle.log(data);
}

fetch('https://example.com/posts/1')

  .catch(error => consle.log(error));

Projekt i zaliczenie

Konfiguracja projektu

  1. Pobieramy i instalujemy Node.js ze strony głównej lub za pomoca NVM
  2. Pobieramy template projektu
  3. Wypakowujemy w miejscu projektu i wpisujemy w konsoli npm install.
  4. W celu startu aplikacji developerskiej wpisujemy w konsoli npm start
  5. W celu zbudowania aplikacji produkcyjnej wpisujemy w konsoli npm run build

Backend do projektu

W projekcie dane naszej aplikacji trzymamy na prostym serwerze plikowym typi-code/json-server.

Serwer uruchamiamy poleceniem npm run server, zostanie on uruchomiony pod adresem http://localhost:3000. Komunikujemy się z nim za pomocą poznanej funkcji fetch()
